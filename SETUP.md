# Setup Guide

Complete prerequisites below, then let Claude Code handle the rest.

## Before You Start

These steps require manual action. Complete them before opening Claude Code.

### HA Instance

- [ ] **SSH access enabled**

  <details>
  <summary><strong>HA OS</strong> (most common)</summary>

  1. Install **Advanced SSH & Web Terminal** add-on (Settings → Add-ons → Add-on Store)
  2. In the add-on configuration, add your SSH public key to `authorized_keys`
  3. Add required packages:
     ```yaml
     packages:
       - rsync
       - bash
     ```
  4. Set **Protection mode** to off (required for filesystem access)
  5. Start the add-on

  </details>

  <details>
  <summary><strong>HA Container / Supervised</strong></summary>

  SSH is already available on your host machine. Ensure:
  - `rsync` is installed on the host
  - Your SSH key can authenticate to the host
  - The HA config directory is accessible (usually `/config/`)

  </details>

- [ ] **Long-lived access token created**
  1. In HA, click your profile (bottom-left)
  2. Scroll to **Long-Lived Access Tokens** → **Create Token**
  3. Name it (e.g., "Claude Code") and copy the token

### Your Machine

- [ ] **Python 3.12+**
  - macOS: `brew install python@3.12`
  - Linux (Debian/Ubuntu): `sudo apt install python3.12 python3.12-venv`
  - Windows: use [WSL2](https://learn.microsoft.com/en-us/windows/wsl/install) with Ubuntu, then follow Linux instructions

- [ ] **rsync 3.x**
  - macOS: `brew install rsync` (system rsync is outdated)
  - Linux: usually pre-installed (`rsync --version` to check)
  - Windows/WSL2: included in Ubuntu WSL

- [ ] **Node.js 20+** *(optional — only needed for the React dashboard)*
  - macOS: `brew install node`
  - Linux: [NodeSource](https://github.com/nodesource/distributions) or `nvm install 20`
  - Skip this if you only want automation templates without the dashboard

- [ ] **Clone and configure**
  ```bash
  git clone https://github.com/dcb/homeassistant-claude-kit
  cd homeassistant-claude-kit
  cp .env.example .env
  ```

  Edit `.env` with your values:
  ```
  HA_HOST=homeassistant.local
  HA_URL=http://homeassistant.local:8123
  HA_TOKEN=your_long_lived_token
  SSH_USER=root
  HA_REMOTE_PATH=/config/
  ```

  > **HA_REMOTE_PATH:** HA OS with SSH add-on uses `/homeassistant/`. HA Container uses `/config/`. If unsure, the setup skill will auto-detect.

  > **No inline comments on value lines.** Write `HA_HOST=192.168.1.100`, not `HA_HOST=192.168.1.100  # my HA`. The Makefile treats everything after `=` as the value, including `# ...` text. Use full-line comments (lines starting with `#`) only.

- [ ] **Install Python tooling**
  ```bash
  make setup
  ```

### Security & Privacy

> **Never commit these files to git:**
> - `.env` -- contains your HA access token and hostname
> - `dashboard/.env.local` -- contains your token (generated by setup skill)
> - `setup-state.json` -- contains your entity mapping answers
>
> The `.gitignore` already excludes them, but verify if you modify it.

> **Privacy mode** (optional): Claude Code reads your config files during sessions. Run `make privacy-on` to block Claude from reading credentials, personal data, and runtime state. See [PRIVACY.md](PRIVACY.md) for full details.

### Optional HA Integrations

Some automation templates and dashboard features depend on additional HA integrations. Install only what you need — the setup skill will skip features for missing integrations.

| Integration | Required for | Install via |
|-------------|-------------|-------------|
| [HACS](https://hacs.xyz/) | Installing the integrations below | [hacs.xyz/docs/use](https://hacs.xyz/docs/use/) |
| [Adaptive Lighting](https://github.com/basnijholt/adaptive-lighting) | Lighting templates (auto color temp/brightness) | HACS |
| [go2rtc](https://github.com/AlexxIT/go2rtc) | Camera streams on dashboard (desktop MSE) | HA Add-on or standalone |
| [WebRTC Camera](https://github.com/AlexxIT/WebRTC) | Camera streams on mobile/iOS (WebRTC) | HACS |
| [Forecast.Solar](https://www.home-assistant.io/integrations/forecast_solar/) | Energy template (solar production forecast) | HA built-in integration |
| `panel_custom` | Dashboard deployment | HA built-in — configured automatically by `make deploy-dashboard` |

> **None of these are required for initial setup.** The setup skills detect what's available and only configure features you have. You can always add integrations later and re-run `setup-customize`.

## From here, Claude handles everything.

Open [Claude Code](https://claude.com/claude-code) in the repo directory and say:

> **"set up my home"**

Two setup skills run automatically:

1. **`setup-infrastructure`** — validates SSH (`$SSH_USER@$HA_HOST`) and API connectivity, auto-detects your config path, installs [ha-ws/ha-api](https://github.com/danbuhler/claude-code-ha) CLI tools on your HA instance, and pulls your configuration.

2. **`setup-customize`** — discovers your entities from HA's floor, area, device, and entity registries. Walks you through a guided interview to map rooms, select automation domains, and configure preferences. Generates the dashboard config files and deploys.

If you prefer to do any step manually, each one is documented below.

## What Claude Does (manual fallback)

Each step below describes what the setup skills automate. The collapsible blocks show how to do it manually — these are for **reference and transparency**, not intended as a complete standalone guide. Some steps (entity discovery, TypeScript generation) are impractical without Claude Code.

### Step 1: Validate API + SSH

Claude tests both connections using `$SSH_USER@$HA_HOST`.

<details>
<summary>Do it manually</summary>

```bash
# Test API
source .env && curl -sf -H "Authorization: Bearer $HA_TOKEN" "$HA_URL/api/" | python3 -m json.tool

# Test SSH
source .env && ssh -o ConnectTimeout=5 "$SSH_USER@$HA_HOST" "echo ok"
```

</details>

### Step 2: Auto-detect config path

Claude checks whether your HA stores config at `/config/` (Container) or `/homeassistant/` (HA OS) and updates `.env` if needed.

<details>
<summary>Do it manually</summary>

```bash
source .env
ssh "$SSH_USER@$HA_HOST" "test -f /homeassistant/configuration.yaml && echo '/homeassistant/' || echo '/config/'"
# Update HA_REMOTE_PATH in .env with the result
```

</details>

### Step 3: Install ha-ws / ha-api CLI tools

Claude clones [claude-code-ha](https://github.com/danbuhler/claude-code-ha) onto your HA instance and runs the install script. These tools enable entity state queries, service calls, and registry management via SSH.

<details>
<summary>Do it manually</summary>

```bash
source .env
ssh "$SSH_USER@$HA_HOST" "cd ${HA_REMOTE_PATH} && git clone https://github.com/danbuhler/claude-code-ha.git"
ssh "$SSH_USER@$HA_HOST" "bash ${HA_REMOTE_PATH}claude-code-ha/install.sh"

# Verify
ssh "$SSH_USER@$HA_HOST" "source /etc/profile.d/claude-ha.sh; source ${HA_REMOTE_PATH}.env; ha-api domains"
```

> **After HA OS updates:** The root filesystem is reset, wiping symlinks and pip packages. The repo persists but you need to re-run the install script:
> `ssh "$SSH_USER@$HA_HOST" "bash ${HA_REMOTE_PATH}claude-code-ha/install.sh"`

</details>

### Step 4: Pull configuration

Claude runs `make pull` to sync your HA config to the local `config/` directory.

<details>
<summary>Do it manually</summary>

```bash
make pull
```

</details>

### Step 5: Discover entities

Claude queries the HA registries via SSH to build a complete map of your entities, rooms, and floors:
- `config/floor_registry/list` — floors
- `config/area_registry/list` — rooms with floor assignments
- `config/device_registry/list` — device-to-area mapping
- `config/entity_registry/list` — entity-to-area overrides

<details>
<summary>Do it manually</summary>

```bash
source .env
ssh "$SSH_USER@$HA_HOST" "source /etc/profile.d/claude-ha.sh; source ${HA_REMOTE_PATH}.env; ha-ws raw config/area_registry/list"
ssh "$SSH_USER@$HA_HOST" "source /etc/profile.d/claude-ha.sh; source ${HA_REMOTE_PATH}.env; ha-ws entity list light"
# Repeat for each domain: binary_sensor, sensor, climate, media_player, camera, etc.
```

Or use the local entity explorer (after `make pull`):
```bash
source venv/bin/activate && python tools/entity_explorer.py --full
```

</details>

### Step 6: Map rooms

Claude walks you through a guided interview, presenting discovered entities by room and asking you to confirm, merge, or skip rooms.

<details>
<summary>Do it manually</summary>

Review `config/.storage/core.area_registry` and `config/.storage/core.device_registry` to understand which entities belong to which rooms. Then manually populate `dashboard/src/lib/areas.ts`.

</details>

### Step 7: Generate entity config

Claude writes `dashboard/src/lib/areas.ts` (room definitions) and `dashboard/src/lib/entities.ts` (200+ entity constants) based on your interview answers.

<details>
<summary>Do it manually</summary>

Copy the template files and fill in your entity IDs by hand. See the inline comments in each file for guidance. This is the step that benefits most from Claude Code — manually mapping 200+ entities is tedious and error-prone.

</details>

### Step 8: Select automation domains

Claude presents a checklist of automation domains (motion lights, climate, energy, health, etc.) and copies the selected templates to `config/automations/`, substituting your entity IDs for the `your_*` placeholders.

<details>
<summary>Do it manually</summary>

```bash
# Copy a template
cp docs/templates/config/automations/lighting.yaml config/automations/lighting.yaml

# Replace placeholders with your entity IDs
# Each template has comments like: "your_room_motion_sensor → actual entity ID"
```

</details>

### Step 9: Build and deploy dashboard

Claude installs npm dependencies, verifies TypeScript compiles, and deploys the dashboard to HA. Skipped if Node.js is not installed.

<details>
<summary>Do it manually</summary>

```bash
cd dashboard && npm install && npm run build && cd ..
make deploy-dashboard
```

</details>

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `401 Unauthorized` on `make push` | Token expired — generate a new one in HA, update `.env` |
| `ha-api` / `ha-ws` not found after HA update | Re-run: `ssh $SSH_USER@$HA_HOST "bash ${HA_REMOTE_PATH}claude-code-ha/install.sh"` |
| `homeassistant.local` doesn't resolve | Use IP address in `.env` instead of hostname |
| Want to connect to a different HA instance | Update `.env`, re-run `setup-infrastructure` skill |
| Setup interrupted mid-way | Re-run `setup-customize` — it resumes from checkpoint |
| `Permission denied` when SSH connects as wrong user | Ensure `SSH_USER=root` in `.env` (not your local username) |
| `make pull` gets wrong files | Check `HA_REMOTE_PATH` — HA OS: `/homeassistant/`, Container: `/config/` |

If your issue isn't listed above, re-run the `setup-infrastructure` skill in Claude Code — it's idempotent and will diagnose and fix most connectivity problems.
